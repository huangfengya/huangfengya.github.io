Git 的基本命令

创建版本库

  1. 初始化 Git 仓库（工作区）

git init

  1. 把文件添加到仓库（暂存区）

git add <file>

可以用 “.” 表示添加所有文件，也可以直接写文件名

注意：如果不把文件添加进入仓库的话，是无法提交该文件的

  1. 提交文件（版本库）

git commit -m ‘说明性文字’

版本回退

使用 git status 命令来查看文件的状态

使用 git diff <file> 命令来查看修改的文件，也可以直接写文件名来查看单独的文件修改

使用 git diff HEAD – <file> 命令来查看工作区和版本库里面最新版本的区别

版本回退

  1. 使用 git log 查看文件提交修改记录(加上 –pretty=oneline 显示简短信息)，对于没有显示的记录，使用 git reflog 查看记录

  2. 使用 git reset –hard 回退到之前的版本(输入 ID 的前几位就可以了)

  3. 原理：HEAD 指向的版本就是当前的版本，通过修改 HEAD 的指向来对版本进行回退

撤销修改（文件误删除）

在未添加到仓库(git add)前，使用 git checkout – <file> 来撤销更改

如果已经添加到仓库(git add)后，使用 git reset HEAD <file> 将添加到库的文件撤回到工作区

分支管理

每次提交，Git 都会把它们串成一条时间线，这条时间线就是一个分支，master 是主分支。

创建及合并分支

  1. git checkout -b dev 创建切换到 dev 分支。相当于命令：

    git branch dev
    git checkout dev

  2. 使用 git branch 查看分支以及当前所在分支；

  3. git checkout master 将分支切换到 master 主分支上；

  4. git merge dev 将 dev 分支上的结果合并到主分支 master 上；

  5. git branch -d dev 删除 dev 分支

  6. 通常，合并分支时，Git 会用 Fast forward 模式，但是这种模式下删除掉分支后，会丢失分支信息。
    如果要强制禁用 Fast forward 模式，Git 就会在 merge 时生成一个新的 commit，这样从分支历史上就可以看出分支信息。 –no-ff
    git merge –no-ff -m ‘merge’ dev

  7. 添加 –graph 参数来查看分支合并情况， git log –graph

解决冲突

当 git merge dev 出现冲突时，使用 git status 查看冲突的文件，修改文件，重新 add/commit/merge 就可以了。

Bug 分支

有时候在进行工作的时候，会出现修改其他bug的问题，使用 git stash 来进行当前状态的保存，创建新分支解决bug并合并。切换到工作分支，使用 git stash list 进行保存状态的查看，使用 git stash apply 将保存的状态恢复，使用 git stash drop 将保存的状态进行删除。使用 git stash pop 切换到当前状态并删除保存状态。

标签管理

创建标签

git tag <tagname> 进行对当前分支打标签， git tag 进行对以前的分支打标签。

使用 git tag 查看所有打的标签。

使用 git show <tagname> 查看此标签所做的更改

标签操作

删除标签：git tag -d <tagname>

推送标签到远程：git push origin <tagname>，git push origin –tags 推送所有标签到远程

删除远程标签：

git tag -d <tagname> // 首先删除本地标签
git push origin :refs/tags/<tagname> // 删除远程标签
