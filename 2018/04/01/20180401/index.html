关于 [1, 2, 3].map(parseInt) 的结果

首先说输出结果：

[1, NaN, NaN]

parseInt() 函数

定义和用法

parseInt() 函数可解析一个字符串，并返回一个整数

语法

parseInt(string, radix)

参数 描述
string 要被解析的值。如果不是字符串，则将其转换为字符串
radix 可选区间为 [2, 36]，用来表示 string 的基数。如未指定，则默认为 10

问：为什么是[2, 36]?

答：因为不存在 0进制和1进制，所以从2进制开始，36 是因为 a 表示10，b 表示 11，z 表示 35，最大是36进制，正好满36进一，不需要新的字母，所以主要原因：英文字母数量不够 (╯‵□′)╯︵┻━┻

返回值

如果被解析参数的第一个字符无法被转换为数值类型，则返回NaN；如果第一个字符转换成数组之后的值大于等于 raidx，返回NaN

parseInt(123, 16) // 返回 1*16^2 + 2*16^1 + 3*16^0 = 256 + 32 + 3 = 291

另外：string在转换字符串的时候，会将大于radix(第一个字母除外)或字母及以后的值删去在进行转换

parseInt(792, 8) // 返回 7，因为 9 > 7，所以讲9及以后的字符串删除

parseInt(75g3, 16) // 返回 117

parsetInt(a1g, 16) // 返回 161，因为 16 进制下 a === 10，g 无法转化成 16 进制内的数，所以删除

map() 函数

定义和用法

map() 创建一个新数组，其结果是该数组中的每个元素都调用一个提供的函数后返回的结果。

语法

arr.map(callback(currentValue, index, array)=> {

  // 处理函数

}[, thisArg])

参数 子参数 描述
callback 生成数组元素的函数，有三个参数
currentVaule callback 的第一个参数，表示数组中正在处理的当前的值
index callback 的第二个参数，表示数组中正在处理的元素的索引
array callback 的第三个参数，表示 map 方法所调用数组
thisArg 可选值。表示执行 callback 函数时使用的 this 值

返回值

返回一个新数组，数组的每个元素都是回调函数的结果

[1, 4, 9].map(Math.sqrt) // 返回 [1, 2, 3]

所以为什么是 [1, NaN, NaN]

首先 parseInt() 接受两个参数，而 map() 提供三个参数，所以 [1, 2, 3].map(parseInt) 实际上等价于

[1, 2, 3].map((currentValue, index)=> {

  return parseInt(currentValue, index)

})

执行过程:

parseInt(1, 0) // 返回 1

parseInt(2, 1) // 2 > 1，返回 NaN

parseInt(3, 2) // 3 > 2，返回 NaN

∴ 结果是 [1, NaN, NaN]
